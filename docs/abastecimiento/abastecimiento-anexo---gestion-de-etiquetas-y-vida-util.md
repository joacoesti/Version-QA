__Tipo__

Anexo operativo técnico

__Uso previsto__

Capacitación, control de etiquetado, vencimientos y soporte operativo de balanza\.

__Alcance__

Productos de Cocina, Milena, Fiambrería, salsas, pastas, ensaladas, empanadas y productos al vacío que pasan por Abastecimiento\.

__Criterio__

No reemplaza definiciones bromatológicas\. Ordena la operación actual, los controles visibles, vida útil y códigos de balanza disponibles\.

__Regla operativa: __Producto sin etiqueta, con vencimiento dudoso o con código incorrecto no debe cruzarse a exhibición\. Primero se corrige, se reetiqueta o se escala\.

__1\. Objetivo operativo__

Estandarizar la gestión de etiquetas, vencimientos y códigos de balanza para que cada producto exhibido o entregado a otro sector tenga identificación visible, fecha coherente y criterio de vida útil claro\. El objetivo no es solamente imprimir una etiqueta: es asegurar que el producto pueda venderse o utilizarse sin dudas para cliente, Abastecimiento, Cocina y Fiambrería\.

__2\. Circuitos actuales de etiquetado__

__Origen__

__Funcionamiento relevado__

__Control esperado__

Cocina

Algunos productos salen etiquetados; otros quedan en cámara para que Abastecimiento los identifique\.

Verificar si llega con etiqueta, fecha y descripción\. Si falta, reetiquetar o pedir corrección\.

Abastecimiento

Etiqueta productos de Cocina que no salen identificados y controla antes de exhibir\.

Imprimir, pegar, revisar vencimiento, código y ubicación visible\.

Milena

Etiqueta pastas frescas y fracciona salsas\. La etiqueta se coloca el día de fraccionamiento\.

Control visual al recibir o reponer: etiqueta visible, fecha coherente y producto correcto\.

Fiambrería

No etiqueta\. Debe recibir producto ya identificado\.

No enviar producto sin etiqueta o con etiqueta dudosa\.

Precios / exhibición

Determinados precios de exhibición se gestionan con referentes administrativos o de precios\.

No modificar precios sin circuito definido\.

__3\. Tipos de etiqueta y datos mínimos__

__Tipo__

__Uso__

__Datos mínimos / criterio__

Etiqueta de balanza

Productos con peso, precio o PLU/código interno: empanadas, salsas, pastas y productos que se pesan o se valorizan en Report NX\.

Descripción, peso/kilaje cuando aplica, fecha de envasado/elaboración, precio, código de barras o PLU\.

Etiqueta superior / impresa

Productos al vacío u otras preparaciones con datos descriptivos\.

Descripción, fecha de elaboración y, cuando aplique, indicación de cocción/preparación\.

Producto unitario

Ejemplo relevado: pollo al spiedo u otros productos vendidos por unidad\.

Puede no llevar kilaje si el producto se gestiona como unidad; debe mantener descripción y fecha según corresponda\.

Producto recibido ya etiquetado

Pastas, salsas o productos fraccionados en otro sector\.

Abastecimiento no duplica etiqueta salvo error: controla que sea correcta y visible\.

__4\. Uso operativo básico de Report NX__

- La balanza tiene productos precargados con código interno, precio unitario o precio por peso según el caso\.
- El operador debe buscar/cargar el código de balanza correspondiente al producto antes de imprimir\.
- Antes de pegar la etiqueta, verificar descripción, peso si aplica, fecha y precio\.
- Si la etiqueta sale mal, falta fecha o se imprime con producto equivocado, se debe reimprimir y reemplazar; no corregir a mano salvo indicación puntual del responsable\.
- Ante falla de máquina, falta de etiquetas o problema de impresión, avisar a Administración o equipo de mantenimiento
- La lista de códigos debe estar visible cerca del punto de trabajo para evitar depender de memoria\.

__5\. Gestión de errores, vencimientos y decomiso__

__Situación__

__Acción esperada__

Etiqueta mal impresa o incompleta

Reimprimir y reemplazar\. Si falta fecha, no cruzar hasta corregir\.

Producto sin etiqueta

No enviar a Fiambrería ni exhibir\. Identificar origen y generar etiqueta si corresponde\.

Producto próximo a vencer

Monitorear hasta fecha límite y decidir uso, retiro o acción según producto\.

Salsas próximas a vencer

Si no se venden, pueden volver a Cocina para uso en producción según criterio operativo relevado\.

Ensaladas vencidas

Descartar mercadería y recuperar/reutilizar plástico si corresponde\.

Producto dañado o estropeado

Separar y pasar por decomiso; no reetiquetar para extender vida útil\.

Decomiso diario

Debe realizarse todos los días como criterio esperado, aunque hoy no siempre se cumple\.

__10\. Checklist operativo rápido__

__Momento__

__Control mínimo__

Antes de etiquetar

Confirmar producto, origen, código de balanza y tipo de etiqueta\.

Al imprimir

Revisar descripción, fecha, peso/precio y legibilidad\.

Antes de cruzar

Controlar etiqueta visible, vencimiento coherente y producto en condiciones\.

Durante reposición

Aplicar FIFO y retirar productos dudosos o próximos a vencer según criterio\.

Ante error

Reimprimir, separar o escalar\. No exhibir producto dudoso\.

Cierre / control

Revisar productos sin etiqueta, próximos a vencer, dañados o para decomiso\.
